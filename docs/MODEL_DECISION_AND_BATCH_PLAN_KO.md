# 모델 결정 + 배치 전략 (고정안)

## 결론

1. 1순위: **Gemini 2.5 Pro**
2. 2순위: **GPT-5 (ChatGPT/Codex)**
3. 비추천: **Gemini Flash 계열**

이유:
- 이번 작업은 속도보다 정확도/누락 방지가 핵심
- 입력 JSONL이 크고, 출력도 거의 같은 크기로 필요
- Flash 계열은 비용/속도 장점은 있지만 정교한 보존형 편집에서 품질 흔들림이 생기기 쉬움

## 컨텍스트/토큰 관점

- 전체 draft는 대략 40만 토큰급 추정이라 단일 턴 처리 위험이 큼
- 출력도 크기 때문에 모델 출력 한도에 걸릴 가능성이 높음
- 따라서 모델과 무관하게 **배치 분할 필수**

## 실행 원칙

1. 배치 자동 생성 (`pack:batches`)
2. 배치별로 같은 프롬프트 사용
3. 결과 파일명 고정: `batch_XX.curated.jsonl`
4. 자동 병합 (`merge:batches`)
5. 자동 검증 (`validate:curated`) 통과 시에만 DB 반영
